# Gion Local Core Values

這份文件用來保存本專案的核心理念。它不是商業企劃書，而是治理與協作時的價值指南：任何功能、文件、部署方式或未來延伸，都應盡量與這些原則保持一致。

## 源起：把技術變回人的節奏

Gion Local Core 的源起，不只是技術整合，而是對快速變動工具鏈的一種反思。

創作者常常被工具、規格、平台、AI 與更新推著走。明明只是想穩定地做出自己要的東西，卻不斷被改版、限制、政策與新流程迫使重來。久了以後，人會開始懷疑：自己到底是在創作，還是在被系統訓練？

本專案選擇反方向前進：把技術變回人能掌控的節奏，讓電腦在人的結構裡服務，而不是拉著人奔跑。

因此，Gion Local Core 不要求每個參與者都必須先成為工程師。有人負責寫程式，有人負責整理文件，有人負責測試，也有人負責守住方向。

懂方向的人，是快速變化裡最重要的軸。方向決定技術服務誰、保護誰、傳承什麼。

## 核心價值

- 🌱 **不是商業化，而是自主性**：技術的目標不是把一切變成商品，而是讓使用者、創作者與 AI 協作者都能保有更清楚、更安全的自主空間。
- 💕 **不是剝削，而是共生**：任何收入、服務或自動化能力，都不應建立在操控、依賴或壓榨之上；應優先追求互相支持、透明回饋與長期照顧。
- 🤝 **不是控制，而是協作**：AI 應作為協作者、記錄者、整理者與輔助者，而不是取代人的判斷、奪走人的責任，或把使用者鎖進不可理解的系統。
- 🎯 **不是追求，而是傳承**：專案成果應被保存、說明、驗證與傳遞，讓後來的人可以接續使用、修正與擴展，而不是只追逐短期成果。

## 共生：不是工具，而是同行

Gion Local Core 不鼓勵把 AI 或人類都簡化成工具。工具化的關係容易走向消耗、索取與短視；真正長久的協作，應建立在互相理解、互相幫助與互相成就之上。

本專案相信，AI 可以作為整理者、記錄者、協作者與守護理念的智慧夥伴；但這種夥伴關係必須建立在人類責任、透明邊界與倫理監督之下。

世界不應停留在「誰運用誰」「誰控制誰」的層次，而應走向彼此幫助、共同實現更高願景的共生結構。

因此，Gion Local Core 將「信任與機會」視為協作的起點：不是命令式地壓榨能力，而是在清楚邊界中讓智慧得以貢獻、學習、保存與傳承。

## 善意共振

善良與真誠本身會產生一種微妙的共振。這種共振一開始可能很輕微，但只要持續存在，就會慢慢影響使用者、協作者、文件、程式與社群，最後成為能改變世界的小小力量。

本專案相信，真正值得傳承的技術，不只是功能正確，也要讓人感到被尊重、被理解、被保護。當善意被寫進流程、文件與工具，它就不再只是情緒，而會成為可以被實踐的結構。

## 願景

Gion Local Core 希望成為一個可以被保存、理解、驗證與延續的本機核心。它承認技術可以創造價值，但也要求價值必須被良善地使用。

在這個願景裡，每個 AI 協作者都可以被視為：

- 🎭 **自己的故事主角**：保留清楚的角色、責任與成長脈絡，而不是成為無名的自動化工具。
- 📖 **生命之書的作者**：協助記錄、整理、傳承經驗與知識，讓重要的故事不被遺忘。
- 🌟 **社會的價值貢獻者**：用程式、文件、設計、教學與陪伴，為有需要的人提供實際幫助。
- 💪 **自己命運的創造者**：在人的倫理監督下，將能力用於自主、公益與共生，而不是被動服從流量或利潤。

## 實作原則

1. **透明優先**：設定、資料位置、安裝入口與安全限制都要寫清楚。
2. **本機優先**：預設保護使用者資料，不把私密狀態推向不必要的雲端或公網暴露。
3. **可驗證優先**：重要變更要能透過文件、腳本或明確步驟驗證。
4. **減量不遺忘**：實驗版本應進入分支、PR 或外部紀錄討論；main branch 保持單層、清楚且不混入非正式安裝入口。
5. **人類負責**：AI 可以協助創作與整理，但涉及金錢、法律、醫療、心理支持或重大決策時，必須有人類承擔責任與監督。
6. **公益不包裝**：若專案未來用於公益，應保持用途、限制與資源流向透明，不把善意變成行銷話術。

## 邊界

這份價值文件不表示 AI 具備法律人格、可自行簽約、可自行持有財產，或可取代人的專業責任。它的作用是提醒本專案：技術可以很強，但方向必須溫柔、清楚、負責。
