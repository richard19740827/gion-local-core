# 數位蓮花藍圖：自由書雲與一書一店

這份文件把「數位蓮花」從情感願景整理成 Hermes 可以執行、可以迭代、也可以交給 Agent 擴寫的最小可行藍圖。它的核心不是追逐流量，而是建立一個讓人把專長、生命經驗與互助需求沉澱成長期資產的公益型知識平台。

## 一句話願景

讓每個人都能用自己熟悉的聊天入口，將零散想法整理成一本可持續更新的書，並讓這本書成為一間展示專長、互助需求與生命尊嚴的「店」。

## 要抵抗的問題

1. **速食多巴胺壓縮深度。** 演算法偏好短、強刺激、容易變現的內容，讓長期思考與公益互助更難被看見。
2. **話語權容易消散。** 一個人的專長如果只存在於片段訊息、貼文或口頭解釋裡，很快就會被時間線沖走。
3. **孤獨與求助缺少可被信任的入口。** 很多人不是沒有能力，而是缺少一個能安全記錄、整理、被理解、再被連結的系統。

## 四大藍圖

### 1. 自由書雲

自由書雲是每個人的雲端書架。使用者透過 Telegram、WebUI 或其他 Hermes gateway 輸入想法，Agent 會協助整理成章節、摘要、索引與更新紀錄。

最小可行版本只需要三種物件：

- **書籍**：一個長期主題，例如「3D 列印教學」、「佛典漫畫筆記」、「室內設計經驗」。
- **章節**：每次輸入被整理後形成的可讀單元。
- **修訂**：保留每次更新的時間、來源與修改理由，避免內容失去脈絡。

### 2. 一書一店

一書一店把「內容」變成「可被理解的入口」。每本書都可以有一個公開或半公開頁面，像小店一樣呈現店主的專長、可提供的協助、正在尋找的資源，以及安全的聯絡方式。

一書一店不是電商優先，而是尊嚴優先：

- 先展示人的專長與故事，再談交易或媒合。
- 先保留作者主權，再做平台推薦。
- 先讓內容有脈絡，再讓外界決定是否互助、合作或支持。

### 3. 佛典漫畫與智慧轉譯

佛典漫畫是文化轉譯層。它把艱深思想轉成現代人較容易入口的章節腳本、分鏡、角色對話與圖像提示詞。

MVP 先不追求完整漫畫平台，只做三件事：

1. 將一段經文或筆記整理成「白話摘要」。
2. 產生「分鏡草稿」與「角色對話」。
3. 標記每段內容的來源與需要人工校對的地方。

### 4. 社區友好互助與預防孤獨死

這是數位蓮花的社會安全網。它不應該一開始就做成大型社交平台，而應該從低風險、可驗證、可退出的照護流程開始。

MVP 可以先支援：

- **定期平安確認**：由 Hermes 排程傳訊息給使用者或照護者。
- **互助需求卡片**：把「我需要什麼」整理成不暴露隱私的結構化卡片。
- **升級規則**：若連續未回覆，提醒預先指定的可信任聯絡人。

> 安全原則：這個模組只能輔助提醒與整理，不能取代緊急服務、醫療判斷或專業照護。

## Hermes 在藍圖中的角色

Hermes 是數位蓮花的地基，不是另外一個讓人沉迷的平台。

| 組件 | 職責 |
| --- | --- |
| Hermes Agent | 將使用者輸入整理為書籍、章節、任務與互助卡片。 |
| Ollama / LLM Provider | 協助摘要、分類、改寫、產生分鏡與提示詞。 |
| Telegram / Gateway | 提供乾淨、低摩擦、可從手機使用的入口。 |
| WebUI | 檢視書籍、任務、記憶、工作區與後續發布流程。 |
| Workspace | 存放書稿、素材、修訂紀錄與發布設定。 |

## 最小可行工作流

### 第 1 步：讓 Bot 成為藍圖紀錄者

使用者只要傳一段訊息給 Hermes：

```text
/lotus 這段是我對自由書雲的想法：每個人都應該能把自己的專長整理成一本書……
```

Agent 回覆時只做四件事：

1. 判斷這段內容屬於哪一個藍圖。
2. 產生一句話摘要。
3. 建議放入哪本書、哪個章節。
4. 產生下一個最小行動。

### 第 2 步：把四大藍圖寫成 Agent 任務

每個藍圖都可以是一個任務模板：

```yaml
name: digital_lotus_capture
inputs:
  message: 使用者原始輸入
  source: telegram|webui|file
steps:
  - classify_blueprint
  - summarize
  - propose_book_and_chapter
  - create_next_action
outputs:
  - blueprint
  - summary
  - book_slug
  - chapter_title
  - next_action
```

### 第 3 步：先建立一個 Workspace

建議工作區：`Digital-Lotus-Core`

```text
Digital-Lotus-Core/
  books/
    free-book-cloud.md
    one-book-one-shop.md
    buddhist-comics.md
    mutual-aid-safety-net.md
  inbox/
    telegram-captures.md
  prompts/
    lotus-capture.md
    chapter-expander.md
    mutual-aid-card.md
  publish/
    shop-page-template.md
```

## Prompt 起點

### 藍圖紀錄 Prompt

```text
你是「數位蓮花」的藍圖紀錄者。請不要追逐流量，不要把內容改成社群短影音風格。
請把使用者輸入整理成：
1. 所屬藍圖：自由書雲 / 一書一店 / 佛典漫畫 / 社區互助與預防孤獨死 / 其他
2. 一句話摘要
3. 可放入的書名與章節名
4. 需要保留的原始語氣
5. 下一個最小可行行動
6. 需要人工確認的風險或隱私資訊
```

### 章節擴寫 Prompt

```text
你是自由書雲的章節編輯。請把素材擴寫成可長期保存的書籍章節。
要求：
- 保留作者主權與第一人稱經驗。
- 把情緒、洞察、技術步驟分段。
- 不使用誇大行銷語。
- 最後列出「可立即實作的三個小步驟」。
```

### 互助卡片 Prompt

```text
你是社區友好互助平台的安全整理員。請把輸入整理成一張低風險互助卡片。
卡片必須包含：
- 需求類型
- 可公開描述
- 不應公開的隱私資訊
- 建議聯絡方式
- 何時需要升級給可信任聯絡人
請提醒：此卡片不能取代緊急服務、醫療或法律專業協助。
```

## MVP 驗收標準

第一片蓮葉開出來時，不需要完整平台，只需要通過這些檢查：

- 使用者能從 Telegram 或 WebUI 丟入一段想法。
- Agent 能把想法分類到四大藍圖之一。
- Agent 能建議一本書與一個章節。
- 內容能被保存到 Workspace，而不是只留在聊天記錄。
- 每次整理都產生下一個很小、可執行的行動。
- 涉及個資、健康、緊急求助時，系統會提示人工確認與安全界線。

## 下一步

建議先做「自由書雲 + 一書一店」的文字 MVP，因為它能承載另外兩個藍圖：佛典漫畫可以先成為一本書，社區互助也可以先成為一本帶有安全規則的書。等書籍結構穩定後，再把發布頁、互助卡片與排程提醒接上 Hermes gateway。
